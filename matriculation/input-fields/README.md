# Input fields

Input fields are simple key-value pairs that let you provide verifiable credential inputs in a format-agnostic way. Define your data once and the credential engine handles the rest, constructing credentials in any format you need.

Input fields are accompanied by

- [translations](translations) - Localization files for Matriculation input fields.
- [input-fields-example](input-fields-example.json) - Input field examples.
- [input-fields-schema](input-fields-schema.json) - JSON Schema definition of input fields.
- [input-fields-to-credential-map](input-fields-to-credential-map.json) - Mapping between user-friendly input fields and the ELM Matriculation fields.
