# PR Review Guidelines Work in progress - template only

This document is for repository maintainers to ensure PRs meet all requirements before approval.

## Checks Before Approval

- Verify all automated tests pass
- Confirm schema changes are valid and versioned
- Ensure README and profiles.md are updated if relevant
- Check that table anchors and links work
- Validate any new examples or data

## Documentation Updates

- Profiles.md updated for new profile or change
- Main README mapping tables updated for new credential types
- Version history / changelog updated if required

## Other Admin Checks

- Code formatting / linting
- PR description clarity
- Proper labels and milestone assignments
